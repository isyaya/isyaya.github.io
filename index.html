函数的四种调用模式

写代码时总是遇到函数调用的问题，今天闲着没事，就总结一下。

当一个函数声明好了之后，是无法确定函数的身份的，只有在调用的时候，通过函数里面的执行情况和结果，才能知道函数的身份。

1.函数调用模式

具体案例：

function fn(a,b){
    return a+b;
    console.log(this);
}
fn();
`</pre>

此时的this指向window,但是以函数的形式调用时通常不使用this关键字。返回值由return语句决定。

2.方法调用模式

把函数作为对象的方法，通过对象来访问这个方法。

具体案例：

<pre>`var obj={
    fn:function(a,b){
        return a+b;
        console.log(this);
    }
}
obj.fn;
`</pre>

该方法内部，this的值指向：调用方法的对象.返回值由return语句决定。

3.构造函数调用模

如果函数或方法调用之前带有关键字new，它就是构造函数调用模式

具体案例：

<pre>`function Person(name,age){
  this.name=name;
  this.age=age;
}
var p1=new Person;
console.log(p1.name);
`</pre>

在该函数内部，this的值指向：构造函数的实例。

如果没有手动添加返回值，返回的就是构造函数的实例;如果手动添加了基本类型的数据作为返回值，返回的还是构造函数的实例，如果手动添加的是对象类型的返回值，最终返回的就是添加的这个对象

4.上下文调用模式

通过call()和apply()间接调用函数，根据用户传递的参数产生不同的结果。

<pre>`function foo(){
        console.log(this.name);
}
foo.call({name:"张三"});//"张三"
foo.call(null);//""
foo.call(3);//undefined
foo.call("ll");//undefined

call方法的第一个参数表示（本次调用该）函数内部的this的值,如果传递null的时候,foo里面的this的值将会变成window。

call方法的第一个参数还可以是数字、字符串、布尔值这些基本类型,此时会转换成基本包装类型。

返回值由return语句决定。

在这里也总结一下call和apply方法的异同：

相同点：

   a.他们都是function.prototype对象中定义的方法

   b.第一个参数是都是函数内部的this的值

不同点：

  如果需要给函数传递参数的时候，利用call方法，将函数的参数从第二个参数开始依次排开；利用apply方法，apply方法的第二个参数是一个数组对象，数组的第一个参数表示函数的第一个实参，依次以此类推。

使用canvas实现数据可视化

绘制折线图

<!DOCTYPE html>
<html>
<head lang="en">
<meta charset="UTF-8">
<title></title>
<style>
    canvas {
        border: 1px solid red;
    }
</style>
</head>
<body>
<canvas id="cas" width="600" height="300"></canvas>
</body>
<script>
var cas=document.getElementById("cas");
var context=cas.getContext('2d');

var points=[
    {x:2,y:97},
    {x:18,y:27},
    {x:21,y:62},
    {x:28,y:44},
    {x:44,y:83},
    {x:61,y:88},
    {x:72,y:45},
    {x:78,y:72},
    {x:84,y:53},
    {x:90,y:42}
];
//定义变量
var leftOffset=20,
        rightOffset=20,
        topOffset=20,
        bottomOffset=20;

//箭头大小
var arrowHeight=20,
        arrowWidth=10;

//坐标轴原点
var x=leftOffset;
var y=cas.height-bottomOffset;

//绘制坐标轴
context.beginPath();
//绘制x轴
context.moveTo(x,y);
context.lineTo(cas.width-rightOffset,y);
//绘制y轴
context.moveTo(x,y);
context.lineTo(x,topOffset);
context.stroke();

//绘制箭头
context.beginPath();
//绘制x轴箭头
context.moveTo(cas.width-rightOffset,y);
context.lineTo(cas.width-rightOffset-arrowHeight,y-arrowWidth/2);
context.lineTo(cas.width-rightOffset-arrowHeight/2,y);
context.lineTo(cas.width-rightOffset-arrowHeight,y+arrowWidth/2);
context.closePath();

//绘制y轴箭头
context.moveTo(x,topOffset);
context.lineTo(x-arrowWidth/2,topOffset+arrowHeight);
context.lineTo(x,topOffset+arrowHeight/2);
context.lineTo(x+arrowWidth/2,topOffset+arrowHeight);
context.closePath();
context.fill();

//绘制点
context.beginPath();
//范围
var rangeWidth=cas.width-leftOffset-rightOffset-arrowHeight;
var rangeHeight=cas.height-topOffset-bottomOffset-arrowHeight;

var maxX=Math.max.apply(null,points.map(function(v){
            return v.x;
        }
));
var maxY=Math.max.apply(null,points.map(function(v){
            return v.y;
        }
));
//比例
var rateX=rangeWidth/maxX;
var rateY=rangeHeight/maxY;

for(var i=0;i<points.length;i++){
    var pointx=x+points[i].x*rateX;
    var pointY=y-points[i].y*rateY;

    context.moveTo(pointx-4,pointY-4);
    context.lineTo(pointx+4,pointY-4);
    context.lineTo(pointx+4,pointY+4);
    context.lineTo(pointx-4,pointY+4);
    context.closePath();
}
context.fillStyle="red";
context.fill();

//绘制折线
context.beginPath();
context.moveTo(x+points[0].x*rateX,pointY=y-points[0].y*rateY)
for(var i=1;i<points.length;i++){
    var pointx=x+points[i].x*rateX;
    var pointY=y-points[i].y*rateY;
    context.lineTo(pointx,pointY);
}
context.strokeStyle="red";
context.stroke();
//绘制坐标
for(var i=0;i<points.length;i++){
    var pointX = x + points[ i ].x * rateX;
    var pointY = y - points[ i ].y * rateY;
    var text="("+points[i].x+","+points[i].y+")";
    context.fillText(text,pointX,pointY);
}
</script>
</html>

继承的几种方式

继承

1 扩展原型对象实现继承

JS继承：将别人的东西拿来自己用

继承关系：s1继承自原型对象 —->s1.proto===原型对象

构造函数有一个prototype属性（默认的原型对象），通过给原型对象添加一个属性或方法，从而可以让构造函数的实例可以访问到，这种继承模型称之为：扩展原型对象实现继承。

2 替换原型对象实现继承

将默认的原型对象替换掉，换成一个全新的对象

为什么要使用替换原型对象实现继承？

当我们需要给构造函数的原型对象中添加很多个属性、方法的时候，如果一直使用扩展对象原型，会多些很多冗余（重复）的代码。

如何实现？

重新给构造函数的prototype属性赋值，值是一个全新的对象，在这个对象中添加属性、方法，注意一定要给这个对象添加constructor属性，值指向构造函数本身。

function Football(){}
Football.prototype={
Constructor:Football,
a1:function(){},
a2:function(){}
}

3 混入继承

使用场景：已知对象o,o2,需要将o中的功能（属性、方法）拷贝到o2中。

实现方式：for…in遍历o1对象的属性，将其属性值赋值到o2中

实现方式：

for(var key in o){//key是字符串类型的值
            //属性的值
     var value=o[key];
//设置o2中的同名属性
o2[key]=value;
}

将混入继承的模型封装成函数：

功能：将源对象中的属性和方法拷贝到目标元素中

target 目标对象：接收数据的对象

source 源对象：数据从那个对象来

function mixin(target,source){
for(var key in source){
var value=source[key];
target[key]=value;
}
return target;
}

应用：jQuery中的$.extend方法利用的就是混入继承的实现原理。

4 原型+混入继承

利用混入继承的原理，向原型对象中添加属性和方法
本质上就是对混入继承的一次应用

function Cat(){}
Cat.prototype.extend=function(source){
mixin(Cat.prototype,source)
};
//mixin(Cat.prototype,{a:10,b:20});
Cat.prototype.extend({a:10,b:20});

原型+混入继承在jQUery中有一个很常见的应用：

$.fn.extend()—>jQuery.prototype.extend()

5 经典继承

蝴蝶书（JavaScript语言精粹）（作者：道格拉斯）

实现功能：已知一个对象o,需要创建一个新的对象，这个对象继承自对象o。

o2.__proto__===o

由于proto不是一个标准的属性，不可适用于在正式代码中，因为它会产生兼容性问题，而且该属性可以被修改。可以删除但是无效。

var o={name:”ll”};
function F(){}
F.prototype=o;
var o2=new F();//当调用构造函数的时候，如果没有参数可以省略括号：new F;

将经典继承封装成函数：

function create(o){
function F(){}
F.prototype=o;
return new F();
}

总结：经典继承的使用场景：要创建一个对象（不需要关心构造函数），新的对象需要继承自另一个指定的对象。

ES5（IE9以下不支持）:Object.create()的实现原理就是源自于经典继承。

在新的浏览器中可以直接用：Object.create（）

在旧的浏览器中应用经典继承：

If(typeof Object.create!==”function”){
Object.create=function(o){
function F(){}
F.prototype=o;
Return new F();
}
}
