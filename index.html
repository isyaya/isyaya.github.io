函数的四种调用模式

写代码时总是遇到函数调用的问题，今天闲着没事，就总结一下。

当一个函数声明好了之后，是无法确定函数的身份的，只有在调用的时候，通过函数里面的执行情况和结果，才能知道函数的身份。

1.函数调用模式

具体案例：

function fn(a,b){
    return a+b;
    console.log(this);
}
fn();
`</pre>

此时的this指向window,但是以函数的形式调用时通常不使用this关键字。返回值由return语句决定。

2.方法调用模式

把函数作为对象的方法，通过对象来访问这个方法。

具体案例：

<pre>`var obj={
    fn:function(a,b){
        return a+b;
        console.log(this);
    }
}
obj.fn;
`</pre>

该方法内部，this的值指向：调用方法的对象.返回值由return语句决定。

3.构造函数调用模

如果函数或方法调用之前带有关键字new，它就是构造函数调用模式

具体案例：

<pre>`function Person(name,age){
  this.name=name;
  this.age=age;
}
var p1=new Person;
console.log(p1.name);
`</pre>

在该函数内部，this的值指向：构造函数的实例。

如果没有手动添加返回值，返回的就是构造函数的实例;如果手动添加了基本类型的数据作为返回值，返回的还是构造函数的实例，如果手动添加的是对象类型的返回值，最终返回的就是添加的这个对象

4.上下文调用模式

通过call()和apply()间接调用函数，根据用户传递的参数产生不同的结果。

<pre>`function foo(){
        console.log(this.name);
}
foo.call({name:"张三"});//"张三"
foo.call(null);//""
foo.call(3);//undefined
foo.call("ll");//undefined

call方法的第一个参数表示（本次调用该）函数内部的this的值,如果传递null的时候,foo里面的this的值将会变成window。

call方法的第一个参数还可以是数字、字符串、布尔值这些基本类型,此时会转换成基本包装类型。

返回值由return语句决定。

在这里也总结一下call和apply方法的异同：

相同点：

   a.他们都是function.prototype对象中定义的方法

   b.第一个参数是都是函数内部的this的值

不同点：

  如果需要给函数传递参数的时候，利用call方法，将函数的参数从第二个参数开始依次排开；利用apply方法，apply方法的第二个参数是一个数组对象，数组的第一个参数表示函数的第一个实参，依次以此类推。

继承的几种方式

继承

1 扩展原型对象实现继承

JS继承：将别人的东西拿来自己用

继承关系：s1继承自原型对象 —->s1.proto===原型对象

构造函数有一个prototype属性（默认的原型对象），通过给原型对象添加一个属性或方法，从而可以让构造函数的实例可以访问到，这种继承模型称之为：扩展原型对象实现继承。

2 替换原型对象实现继承

将默认的原型对象替换掉，换成一个全新的对象

为什么要使用替换原型对象实现继承？

当我们需要给构造函数的原型对象中添加很多个属性、方法的时候，如果一直使用扩展对象原型，会多些很多冗余（重复）的代码。

如何实现？

重新给构造函数的prototype属性赋值，值是一个全新的对象，在这个对象中添加属性、方法，注意一定要给这个对象添加constructor属性，值指向构造函数本身。

function Football(){}
Football.prototype={
Constructor:Football,
a1:function(){},
a2:function(){}
}

3 混入继承

使用场景：已知对象o,o2,需要将o中的功能（属性、方法）拷贝到o2中。

实现方式：for…in遍历o1对象的属性，将其属性值赋值到o2中

实现方式：

for(var key in o){//key是字符串类型的值
            //属性的值
     var value=o[key];
//设置o2中的同名属性
o2[key]=value;
}

将混入继承的模型封装成函数：

功能：将源对象中的属性和方法拷贝到目标元素中

target 目标对象：接收数据的对象

source 源对象：数据从那个对象来

function mixin(target,source){
for(var key in source){
var value=source[key];
target[key]=value;
}
return target;
}

应用：jQuery中的$.extend方法利用的就是混入继承的实现原理。

4 原型+混入继承

利用混入继承的原理，向原型对象中添加属性和方法
本质上就是对混入继承的一次应用

function Cat(){}
Cat.prototype.extend=function(source){
mixin(Cat.prototype,source)
};
//mixin(Cat.prototype,{a:10,b:20});
Cat.prototype.extend({a:10,b:20});

原型+混入继承在jQUery中有一个很常见的应用：

$.fn.extend()—>jQuery.prototype.extend()

5 经典继承

蝴蝶书（JavaScript语言精粹）（作者：道格拉斯）

实现功能：已知一个对象o,需要创建一个新的对象，这个对象继承自对象o。

o2.__proto__===o

由于proto不是一个标准的属性，不可适用于在正式代码中，因为它会产生兼容性问题，而且该属性可以被修改。可以删除但是无效。

var o={name:”ll”};
function F(){}
F.prototype=o;
var o2=new F();//当调用构造函数的时候，如果没有参数可以省略括号：new F;

将经典继承封装成函数：

function create(o){
function F(){}
F.prototype=o;
return new F();
}

总结：经典继承的使用场景：要创建一个对象（不需要关心构造函数），新的对象需要继承自另一个指定的对象。

ES5（IE9以下不支持）:Object.create()的实现原理就是源自于经典继承。

在新的浏览器中可以直接用：Object.create（）

在旧的浏览器中应用经典继承：

If(typeof Object.create!==”function”){
Object.create=function(o){
function F(){}
F.prototype=o;
Return new F();
}

javas模块化开发

1. 模块化

模块的作用就是：私有空间；可以加载，可以导出

  • 什么是模块化？

    • 模块化是指解决一个复杂问题时自顶向下逐层把系统划分成若干模块的过程，有多种属性，分别反映其内部特性
    • 解决复杂问题的一种方式

  • 使用模块化带来的好处

    • 生产效率高
    • 可维护性高

2. 模块化开发演变

2.1 全局函数

  • 污染了全局变量
  • 模块成员之间看不出直接联系

2.2 命名空间

  • 理论意义上减少了变量冲突
  • 缺点1：暴露了模块中所有的成员，内部状态可以被外部改写，不安全
  • 缺点2：命名空间会越来越长

2.3 私有空间

  • 私有空间的变量和函数不会影响全局作用域
  • 公开共有方法，隐藏私有属性

2.4 匿名函数自执行的特性

如果不使用第三方规范的情况下，如果写模块可以采用下面这种方式：

  • ；(function(){})()
    • 匿名自执行函数加分号的作用是为了防止前面的代码没有加分号造成语法解析错误
    • 将代码放到一个匿名自执行函数中是为了避免全局命名空间污染，核心就是利用函数的私有作用域
    • 将依赖项作为参数传递到匿名函数自执行的目的是为了减少作用域查找机制，提高代码执行效率

3. 模块化规范（Sea.js）

Seajs：一个JavaScript模块加载器，可以用一个js文件加载另一个js文件

Sea.js 可以解决命名空间污染、文件依赖的问题。

模块具有两个特性：

  • 模块要有一个私有作用于：避免全局命名空间污染

  • 模块可以向外导出内部成员，供别的模块加载和使用

要构建一个系统，最基本的层面需要做两件事情：

3.1 定义系统成员：

确定成员是什么

  • 模块是一个JavaScript文件
  • 每一个模块都是用define函数去定义

3.2 约定系统通讯：

确定成员之间如何交互，遵循规则是什么

  • 一个SeaJS模块默认就是私有作用域
  • 如果想要被外部文件模块访问，就必须把要公开的属性挂载给module.exports 对象接口
  • 使用require 函数可以加载一个指定的模块，得到该模块代码中暴露的接口对象

3.3 如何启动整个模块系统

  • 在html页面中使用seajs.use()
    方法，指定一个入口文件模块

define方法的回调函数中分别传递三个参数：

define（function（require,exports,modules）{
console.log(‘main.js文件被加载了’);
require(‘./a’);
}）

使用seajs.use方法，指定一个模块，启动整个模块系统

seajs.use(../js/main);

加载的模块路径可以省略后缀名

每一个模块中有一个require 函数，可以用来加载指定模块，需要接受一个模块路径

模块的作用域和导出：

模块天生就是一个私有作用域，在该模块内部定义的所有成员，默认都是该模块的私有变量。如果模块内部的成员想要被外部所访问：必须通过module.exports对象向外暴露

函数加载模块的时候，会自动拿到模块内部的module.exports对象

exports:SeaJs提供的接口对象，exports是module.exports接口对象的一个引用，也就是说：修改了exports相当于修改了module.exports,但是如果想要向外暴露一个单独的变量、函数等成员，那就必须通过给modules.exports赋值才行，在模块内部给xports赋值不管用

加载模块，启动模块系统。

  • 加载一个模块seajs.use('id')
  • 加载一个模块，在加载完成时，执行回调seajs.use('id',callback)
  • 家在多个模块，加载完成时执行回调 sejs.use(['id1','id2'],callback)

4.2 define

define 是一个全局函数，用来定义模块。

define接受参数，参数可以是一个函数，也可以是一个对象或字符串。

参数为对象、字符串是，表示模块的接口就是对象、字符串。

4.3 require

require 用来加载一个js文件模块，Require用来获取指定模块的接口对象 modele.exports。

require 在加载和执行的时候，js会按照同步的方式执行。
