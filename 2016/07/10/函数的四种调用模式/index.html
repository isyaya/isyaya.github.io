函数的四种调用模式

写代码时总是遇到函数调用的问题，今天闲着没事，就总结一下。

当一个函数声明好了之后，是无法确定函数的身份的，只有在调用的时候，通过函数里面的执行情况和结果，才能知道函数的身份。

1.函数调用模式

具体案例：

function fn(a,b){
    return a+b;
    console.log(this);
}
fn();
`</pre>

此时的this指向window,但是以函数的形式调用时通常不使用this关键字。返回值由return语句决定。

2.方法调用模式

把函数作为对象的方法，通过对象来访问这个方法。

具体案例：

<pre>`var obj={
    fn:function(a,b){
        return a+b;
        console.log(this);
    }
}
obj.fn;
`</pre>

该方法内部，this的值指向：调用方法的对象.返回值由return语句决定。

3.构造函数调用模

如果函数或方法调用之前带有关键字new，它就是构造函数调用模式

具体案例：

<pre>`function Person(name,age){
  this.name=name;
  this.age=age;
}
var p1=new Person;
console.log(p1.name);
`</pre>

在该函数内部，this的值指向：构造函数的实例。

如果没有手动添加返回值，返回的就是构造函数的实例;如果手动添加了基本类型的数据作为返回值，返回的还是构造函数的实例，如果手动添加的是对象类型的返回值，最终返回的就是添加的这个对象

4.上下文调用模式

通过call()和apply()间接调用函数，根据用户传递的参数产生不同的结果。

<pre>`function foo(){
        console.log(this.name);
}
foo.call({name:"张三"});//"张三"
foo.call(null);//""
foo.call(3);//undefined
foo.call("ll");//undefined

call方法的第一个参数表示（本次调用该）函数内部的this的值,如果传递null的时候,foo里面的this的值将会变成window。

call方法的第一个参数还可以是数字、字符串、布尔值这些基本类型,此时会转换成基本包装类型。

返回值由return语句决定。

在这里也总结一下call和apply方法的异同：

相同点：

   a.他们都是function.prototype对象中定义的方法

   b.第一个参数是都是函数内部的this的值

不同点：

  如果需要给函数传递参数的时候，利用call方法，将函数的参数从第二个参数开始依次排开；利用apply方法，apply方法的第二个参数是一个数组对象，数组的第一个参数表示函数的第一个实参，依次以此类推。
